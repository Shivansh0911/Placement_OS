## Prompts Used

### Prompt 1 — Resume Screening Agent

```
Build a production-quality FastAPI + LangChain resume screening agent for campus placement drives.

Requirements:
- POST /screen accepts job_description, eligibility_criteria (dict), and a list of resumes (student_id, name, resume_text)
- Each resume is scored 0-100 against the JD using Gemini 1.5 Flash (free tier)
- Returns a ranked JSON list with: score, reasoning, strengths (list), gaps (list), recommendation (STRONG_YES/YES/MAYBE/NO)
- Process resumes concurrently with asyncio.gather, batch size 10
- Add exponential backoff retry (3 attempts) for OpenAI RateLimitError and APITimeoutError
- If a single resume fails to parse, return a neutral MAYBE score with a note — don't crash the batch
- GET /health returns service status and whether OPENAI_API_KEY is set
- Use Pydantic v2 for all models
- Prompts in a separate prompts.py — system prompt should emphasize calibration: most campus candidates should score 40-70, reserve 85+ for exceptional profiles
- Include render.yaml for free-tier Render.com deployment

Output files: main.py, agent.py, models.py, prompts.py, requirements.txt, render.yaml
```

### Prompt 2 — Mock College ERP

```
Build a FastAPI mock REST API simulating a college ERP system for campus placement data.

Endpoints:
- GET /students/eligible — filter by min_cgpa (float), branches (comma-separated string), max_backlogs (int)
- GET /students/{student_id} — get single student
- POST /placements/record — write a placement record (drive_id, company_name, role, package_lpa, student_id, student_name, joining_date)
- GET /placements — list all placements, filterable by drive_id and company_name
- GET /placements/summary — total offers, companies, avg package
- DELETE /placements/reset — clear all records (demo use)

Data requirements:
- 20 realistic Indian engineering students from BITS Pilani Hyderabad
- Mix of branches: CSE (8), IT (3), ECE (4), EEE (2), Mechanical (2), Civil (1)
- CGPAs from 6.2 to 9.3
- 3 students with backlogs (1 with 1 backlog, 1 with 2 backlogs)
- Resume text per student: specific, realistic, includes internship company names, 
  project names with outcomes, skills, achievements — NOT generic buzzwords

Store data in JSON files. Include render.yaml for Render.com deployment.

Output: main.py, data/students.json, requirements.txt, render.yaml
```

1. Open a terminal with Claude Code running
2. Paste Prompt 1 and show it generating the files in real-time

5. Show a test POST to `/screen` with a sample payload
6. Narrate: *"This entire screening agent was generated by Claude Code. No boilerplate written manually."*
